LMS Project Guide (Client Handover)

This document is written for non-developers as well as technical staff. It explains how to install and run the project (with exact commands), how the system works, and what you can do in the admin, teacher, and student areas.

Project Type: Laravel 10 Learning Management System (LMS) — PHP + MySQL (or compatible database)

Part A — What You Must Install First (One Time)

Think of these as the “engines” that run the project on your computer or server.

SoftwareWhy you need itWhere to get it
PHP (8.1 or newer) Laravel runs on PHP. windows.php.net or use XAMPP / Laragon (includes PHP + MySQL).
Composer Downloads PHP libraries for the project. getcomposer.org
MySQL (or MariaDB) Database where users, courses, orders, etc. are stored. Included in XAMPP/Laragon, or install MySQL Server separately.
Node.js (optional for local dev) Only needed if you change front-end assets with Vite (npm run dev). nodejs.org (LTS version).
Tip: On Windows, Laragon or XAMPP is often easiest: you get PHP, MySQL, and phpMyAdmin in one package. After installing, add PHP and Composer to your system PATH so commands work in PowerShell or Command Prompt.

Part B — First-Time Project Setup (Step by Step + Commands)

Important: Run all commands from the project root folder (the folder that contains artisan, composer.json, and .env.example). In this guide we use D:\laraval\LMS as an example — change it to your actual folder path.

B1. Open a terminal in the project folder

  • Windows: Open PowerShell → cd D:\laraval\LMS
  • Mac/Linux: Open Terminal → cd /path/to/LMS

B2. Install PHP dependencies (Composer)

This downloads Laravel and all required packages. Run once after cloning or copying the project.

composer install

If Composer is not recognized, install it from getcomposer.org and try again.

B3. Create your environment file (.env)

Laravel reads database and app settings from .env. If you do not have .env yet, copy the example file:

Windows (PowerShell):

Copy-Item .env.example .env

Mac/Linux:

cp .env.example .env

B4. Generate application key

Required for encryption and sessions.

php artisan key:generate

B5. Create an empty database in MySQL

Using phpMyAdmin, MySQL Workbench, or command line, create a database, for example named lms_db. Note the database name, username, and password.

B6. Edit .env — connect the database

Open .env in a text editor and set at least these lines (use your real values):

APP_NAME=LMS APP_URL=http://127.0.0.1:8000 DB_CONNECTION=mysql DB_HOST=127.0.0.1 DB_PORT=3306 DB_DATABASE=lms_db DB_USERNAME=root DB_PASSWORD=

Leave DB_PASSWORD empty if your MySQL user has no password (common on local XAMPP).

B7. Run database migrations

This creates all tables (users, courses, orders, assignments, etc.).

php artisan migrate

If you see “Nothing to migrate”, tables may already exist — that is OK.

B8. Seed default admin user (optional but recommended)

Creates the default admin account (see login section below).

php artisan db:seed

B9. Link storage for uploaded files (images, payment proofs)

Makes files in storage/app/public visible on the website.

php artisan storage:link

B10. Clear config cache (after changing .env)

php artisan config:clear php artisan cache:clear

B11. Start the local web server

php artisan serve

Then open a browser and go to: http://127.0.0.1:8000

Admin dashboard URL (after login as admin): typically http://127.0.0.1:8000/admin/dashboard (exact path depends on your routes; use “Dashboard” after logging in).

B12. (Optional) Front-end build with Node

Only if you modify JavaScript/CSS through Vite:

npm install npm run build

For live reloading while developing: npm run dev (run in a second terminal).

Part D — How the Project Works (Simple Overview)

  • Public website — Visitors see home, course catalog, course details, cart, checkout, login/register. Some menu items may be hidden in the header but pages can still exist if you open the URL directly.
  • Checkout & payments — Students pay (e.g. upload payment proof). Admins review orders under Payment proofs and can set Payment methods (account numbers and instructions) from the admin panel.
  • After purchase — The student is enrolled in courses and can open lessons, quizzes, and assignments from the student dashboard.
  • Teachers — Manage their courses (as configured), assignments, and monitor students linked to their teaching.
  • Admins — Manage users, courses, categories, content, transactions, reviews, announcements, Q&A, discussions, assignments analytics, and more from the sidebar.

Technical folders (for your developer):

  • routes/web.php — URL routes.
  • app/Http/Controllers — Logic for pages and actions.
  • resources/views — HTML templates (Blade).
  • database/migrations — Database table definitions.
  • config/payment.php — Default payment method labels/numbers (admin can override in Payment methods).

Part E — What You Can Do in This System (By Role)

As Admin (after login)

  • Dashboard — See totals: students, courses, earnings, enrollments, assignment activity.
  • Users — Add teachers/students, approve or reject accounts if your workflow uses that.
  • Course categories & courses — Create and publish courses, set prices, assign teachers.
  • Payment proofs — Approve or reject manual payment uploads; filter by status and gateway.
  • Payment methods — Edit display names, account numbers, and instructions shown at checkout.
  • Transactions — View payment records.
  • Assignments (menu) — Manage all assignments, teacher activity, student performance.
  • Quizzes, reviews, announcements, Q&A, discussions — Moderation and content as needed.
  • Profile — Update admin profile.
Some sidebar sections (e.g. blog, zoom, FAQ management) may be commented out in code for your build—your developer can turn them back on if you need them visible.

As Teacher

  • View dashboard metrics relevant to teaching and assignments.
  • Create and grade assignments for courses you are allowed to manage.
  • Monitor students’ progress for your courses (where enabled).
  • Use Q&A, discussions, announcements as configured.

As Student

  • Browse and purchase courses (or enroll in free courses).
  • Open Enrolled Courses, complete lessons and quizzes in order where the platform enforces progression.
  • Submit assignments and view feedback/grades.
  • See announcements and notifications.

Part H — Key User Journeys

H1. From visitor to enrolled student

  1. Browse courses on the public site → open a course → add to cart.
  2. Register or log in.
  3. Checkout: choose payment method, upload proof if required, submit.
  4. Admin approves payment (if pending verification) → student becomes enrolled.
  5. Student opens Enrolled Courses and starts lessons.

H2. Assignments

  1. Teacher or admin creates an assignment for a course (title, description, deadline).
  2. Student submits text or files.
  3. Teacher or admin grades and adds feedback.
  4. Admin can use assignment reports to see workload and results.

Part I — Important Admin URLs (Reference)

  • /admin/assignments/manage — All assignments.
  • /admin/assignments/teacher-activity — Teacher workload.
  • /admin/assignments/performance — Student progress and marks.
  • /admin/payment-orders — Payment proof queue (pending / approved / rejected).
  • /admin/payment-methods — Edit checkout payment text and account numbers.

Your site base URL is added in front (e.g. http://127.0.0.1:8000/admin/payment-orders).

Part J — Dashboard Numbers (Plain Language)

  • Enrolled courses — How many course enrollments exist.
  • Active / completed courses — Progress-based totals.
  • Assignments / submissions — Work created and handed in.
  • Earnings — Sum from paid orders (admin view).

Part K — Technical Reference (For Developers)

DashboardController

  • Builds admin dashboard metrics: enrollments, courses, earnings, plus assignment-related counts where applicable.

TeacherAssignmentController

  • Lists/creates assignments, shows detail, grades submissions, monitors student progress for allowed courses.

StudentAssignmentController

  • Student list, detail, and submit actions for enrolled courses.

AssignmentAdminController

  • Admin-only: global assignment list, teacher activity, student performance matrix.

Part L — Troubleshooting

ProblemWhat to try
SQLSTATE / database connection error Check MySQL is running; verify DB_* in .env; run php artisan config:clear.
500 error / white page Run php artisan key:generate; set APP_DEBUG=true locally to see details (turn off in production).
Images or uploads not showing Run php artisan storage:link; check folder permissions on storage and bootstrap/cache.
composer or php not found Install PHP and Composer; add them to your system PATH; restart the terminal.

Part M — Production (Going Live) Checklist

  • Set APP_ENV=production and APP_DEBUG=false in .env.
  • Use a strong APP_KEY (already set by key:generate).
  • Use real database credentials and HTTPS URL in APP_URL.
  • Run composer install --no-dev --optimize-autoloader on the server.
  • Run php artisan migrate --force (after backup).
  • Run php artisan config:cache and php artisan route:cache when your developer recommends it.
  • Change the default admin password.
