LMS Project Guide (Client Handover)

This document explains how your Laravel LMS project works: modules, user roles, key functions, dashboards, assignments flow, and admin access details.

Project Type: Laravel-based Learning Management System (LMS)

1) What This System Does

  • Public website for marketing pages (home, courses, blogs, FAQs, contact, about, zoom, membership).
  • User authentication (register/login/logout) and role-based dashboards.
  • Course purchase + enrollment + student learning journey (sections, lessons, quiz flow).
  • Teacher/admin assignment creation, grading, and student progress monitoring.
  • Admin controls for users, courses, reviews, quizzes, transactions, announcements, Q&A, discussions, and assignment analytics.

4) High-Level Architecture

  • Routes: routes/web.php defines public, auth, admin, teacher, and student routes.
  • Controllers: Business logic in app/Http/Controllers/Admin and app/Http/Controllers/Website.
  • Models: Entities like User, Course, Assignment, AssignmentSubmission.
  • Views: Blade templates in resources/views, including dashboard and assignment pages.
  • Database: Core tables include users, courses, sections, lessons, quizzes, attempts, assignments, submissions, and pivot/enrollment tables.

5) Key Functional Flow (End-to-End)

A. Public to Purchase

  • User browses courses/blog pages.
  • User registers/logs in.
  • User adds courses to cart and checks out.
  • System records order and enrollment.

B. Learning Flow

  • Student opens enrolled course.
  • Moves through sections and lessons.
  • Watch progress and quiz pass rules unlock progression.
  • Student can submit assignments where available.

C. Teaching/Admin Flow

  • Teacher/Admin creates assignments per course.
  • Students submit text/file responses.
  • Teacher/Admin reviews submissions and grades.
  • Admin monitors teacher activity and platform-wide student performance.

6) Important Controllers and What Their Functions Do

DashboardController (Admin Area)

  • index(): builds dashboard metrics (enrollments, active/completed courses, totals, earnings).
  • Also includes assignment metrics for admin/teacher:
    • Assigned courses
    • Created assignments
    • Student submissions

TeacherAssignmentController

  • __construct(): allows only teacher/admin roles.
  • teacherCourseIds(): resolves allowed course IDs for current user.
  • assertCourseAllowed()/assertAssignmentAllowed(): authorization guards.
  • index(): list assignments (teacher-owned or all for admin scope through this controller).
  • create(): loads valid courses for assignment creation form.
  • store(): validates and saves assignment.
  • show(): assignment detail + enrolled students + submission status.
  • gradeSubmission(): saves marks/feedback and graded timestamp.
  • monitor(): per-course student lesson and quiz progress view.

StudentAssignmentController

  • Handles student assignment list, detail page, and submission action for enrolled courses.

AssignmentAdminController (Admin-only Controls)

  • manage(): paginated global list of all assignments with submission and graded counts.
  • teacherActivity(): teacher-level workload metrics (courses, assignments, recent activity, pending grading).
  • performance(): student progress matrix by course (lessons done, quizzes passed, average assignment marks).

7) Assignment Module (Detailed)

StageSystem Behavior
Create Assignment Teacher/Admin chooses course, title, description, deadline. Validation ensures valid course and permissions.
Student Submission Student submits text/file for assignment. Submission timestamp is stored.
Grading Teacher/Admin enters marks and feedback; system stores grade and graded timestamp.
Admin Monitoring Admin can view all assignments, teacher output, and student progress/performance analytics.

8) Admin Assignment Pages Added

  • /admin/assignments/manage - Manage all assignments.
  • /admin/assignments/teacher-activity - Monitor teacher activity.
  • /admin/assignments/performance - View student results and progress.

Sidebar includes an Assignments dropdown with these pages plus teacher workflow links.

9) Dashboard Cards Explained (Client Language)

  • Enrolled Courses: Total purchased/enrolled course records.
  • Active Courses: Enrolled but not fully completed.
  • Completed Courses: Courses where lessons are completed according to platform rule.
  • Assigned Courses: Courses currently tied to assignment operations.
  • Created Assignments: Number of assignment records created.
  • Student Submissions: Number of submitted assignment responses.
  • Total Students/Courses/Earnings: Global admin business metrics.
